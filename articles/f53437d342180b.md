## やりたいこと

CursorやVSCodeでは、Dockアイコンを右クリックすると最近開いたフォルダの一覧が表示されます。

![Dockの右クリックメニュー](/images/cursor-vscode-shortcut-app/dock-recent.png)

これはとても便利なのですが、たくさんのプロジェクトに関わっていると、この履歴だけでは足りません。久しぶりに開くフォルダは「ファイル → フォルダを開く」から探すしかなく、地味に面倒です。

表示件数はカスタマイズできなさそうなので、**ダブルクリック一発で指定フォルダをCursor/VSCodeで開けるショートカットアプリ**を作ることにしました。これでサクサクアクセスできます。

![作成したショートカットアプリ](/images/cursor-vscode-shortcut-app/finder-apps.png)

## 方法：AppleScriptで `.app` を作る

macOSの `osacompile` コマンドを使って、AppleScriptをアプリケーション（`.app`）にコンパイルします。

### Cursorの場合

```bash
osacompile -o ~/Desktop/my-project.app \
  -e 'do shell script "open -a '\''Cursor'\'' ~/workspace/my-project"'
```

### VSCodeの場合

```bash
osacompile -o ~/Desktop/my-project.app \
  -e 'do shell script "open -a '\''Visual Studio Code'\'' ~/workspace/my-project"'
```

各パラメータの意味は以下のとおりです：

- `-o ~/Desktop/my-project.app` … 出力先のファイルパス（ファイル名や場所は自由に変更可）
- `-e '...'` … コンパイルするAppleScriptの内容
- `open -a 'Cursor'` … Cursorアプリで開く
- `~/workspace/my-project` … 開きたいフォルダのパス

たったこれだけです。デスクトップに `.app` ファイルが生成されます。

## 仕組み

`osacompile` は AppleScript をコンパイルして `.app`（アプリケーションバンドル）を生成するmacOS標準のコマンドです。

生成された `.app` の中身は以下のAppleScriptです：

```applescript
do shell script "open -a 'Cursor' ~/workspace/my-project"
```

`open -a` はmacOSのコマンドで、指定したアプリケーションでファイルやフォルダを開きます。Cursorはフォルダを渡すとそのフォルダをワークスペースとして開いてくれます。

## 一括作成する

プロジェクトが複数ある場合、シェルスクリプトで一括作成できます。

```bash
#!/bin/zsh

# アプリ名とパスの対応を定義
declare -A projects=(
  ["my-app"]="$HOME/workspace/my-app"
  ["api-server"]="$HOME/workspace/api-server"
  ["frontend"]="$HOME/workspace/frontend"
)

# エディタを指定（"Cursor" または "Visual Studio Code"）
EDITOR_APP="Cursor"

# 出力先フォルダ
OUTPUT_DIR="$HOME/Desktop/editor-shortcuts"
mkdir -p "$OUTPUT_DIR"

# 一括作成
for name path in "${(@kv)projects}"; do
  osacompile -o "$OUTPUT_DIR/$name.app" \
    -e "do shell script \"open -a '$EDITOR_APP' '$path'\""
  echo "Created: $name.app → $path"
done
```

## 活用Tips

### Dockに追加する

作成した `.app` をDockにドラッグ&ドロップするだけで、ワンクリックで起動できるようになります。

### Spotlight / Raycastから起動する

`.app` ファイルはSpotlightやRaycastの検索対象になります。ファイル名を分かりやすくしておけば、キーボードだけで素早く起動できます。

### ファイル名・置き場所は自由

`.app` の中にパスが埋め込まれているため、ファイル名の変更や別フォルダへの移動をしても動作に影響しません。

## `.command` ファイルではダメなの？

`.command` ファイル（シェルスクリプト）でも同じことはできますが、実行するとターミナルウィンドウが開いたまま残ってしまいます。AppleScriptの `.app` ならターミナルは一切表示されません。

| 方法                  | ターミナル | Dock追加 | Spotlight対応 |
| --------------------- | ---------- | -------- | ------------- |
| `.command`            | 残る       | △        | ×             |
| `.app`（AppleScript） | 開かない   | ○        | ○             |

## まとめ

- `osacompile` でAppleScriptを `.app` にすれば、ダブルクリックでCursor/VSCodeが起動する
- ターミナルが開かないのでストレスフリー
- Dock・Spotlight・Raycastとも相性が良い
- シェルスクリプトで一括作成も簡単
