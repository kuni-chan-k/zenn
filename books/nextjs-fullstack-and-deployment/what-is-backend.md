## フロントエンドとバックエンド

Webアプリケーションは、大きく2つの部分に分かれます。

```
┌─────────────────────────────────────────────────┐
│                  Webアプリケーション               │
├────────────────────┬────────────────────────────┤
│   フロントエンド    │       バックエンド          │
│   （クライアント）   │       （サーバー）          │
├────────────────────┼────────────────────────────┤
│ ・画面の表示        │ ・データの保存・取得        │
│ ・ユーザー操作      │ ・ビジネスロジック          │
│ ・見た目の制御      │ ・認証・セキュリティ        │
│                    │ ・外部サービス連携          │
└────────────────────┴────────────────────────────┘
```

### フロントエンド（クライアント）

ユーザーが直接触れる部分です。

- ブラウザで動作する
- 画面の表示、ボタンのクリック、フォーム入力など
- React、Next.js で作る部分

「Reactで作る！フロントエンド完結アプリ開発」で学んだのは、主にフロントエンドです。

### バックエンド（サーバー）

ユーザーからは見えない部分です。

- サーバーで動作する
- データの保存、認証、セキュリティなど
- API、データベース、認証システムなど

「Next.jsでバックエンド連携＆サービス公開」で学ぶのは、バックエンドです。

## なぜバックエンドが必要？

「Reactで作る！フロントエンド完結アプリ開発」のPart 3で作ったアプリは、localStorage を使ってデータを保存しました。

これには以下の制限があります。

| 制限 | 説明 |
|-----|------|
| ブラウザ限定 | データは1つのブラウザにしか保存されない |
| 共有不可 | 他のユーザーとデータを共有できない |
| 消失リスク | ブラウザのデータを消すとすべて消える |
| セキュリティ | 誰でもデータを見られる・改ざんできる |

バックエンドを使うと、これらの制限を解決できます。

| バックエンドで解決 | 説明 |
|------------------|------|
| どこからでもアクセス | スマホ、PC、どのデバイスからでも同じデータ |
| データ共有 | 複数ユーザーで同じデータを共有 |
| データ永続化 | サーバーに保存されるので消えない |
| セキュリティ | 認証・認可でデータを保護 |

## クライアント・サーバーモデル

バックエンドとフロントエンドは、**リクエスト**と**レスポンス**でやり取りします。

```
┌──────────────┐                    ┌──────────────┐
│              │   ① リクエスト     │              │
│  クライアント  │ ─────────────────→ │    サーバー   │
│  (ブラウザ)   │                    │  (バックエンド) │
│              │ ←───────────────── │              │
│              │   ② レスポンス     │              │
└──────────────┘                    └──────────────┘
```

1. **リクエスト**: クライアントがサーバーにデータを要求
2. **レスポンス**: サーバーがクライアントにデータを返す

例えば、大会一覧を表示する場合：

1. ブラウザが「大会一覧をください」とリクエスト
2. サーバーがデータベースから大会を取得
3. サーバーが大会一覧をレスポンスとして返す
4. ブラウザが受け取ったデータを画面に表示

## Next.js でのバックエンド

Next.js では、**API Routes** という機能でバックエンドを作れます。

```
my-app/
├── app/
│   ├── page.tsx          ← フロントエンド（画面）
│   └── api/
│       └── competitions/
│           └── route.ts  ← バックエンド（API）
```

フロントエンドとバックエンドを**同じプロジェクト**で開発できるのが Next.js の特徴です。

## この章のまとめ

- Webアプリはフロントエンド（見える部分）とバックエンド（見えない部分）に分かれる
- バックエンドはデータの保存、認証、セキュリティなどを担当
- クライアントとサーバーはリクエストとレスポンスでやり取りする
- Next.js では API Routes でバックエンドを作れる
