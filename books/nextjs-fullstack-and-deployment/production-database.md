## この章でやること

- 本番用データベースの選択肢を知る
- Vercel Postgres をセットアップする
- 本番環境でのマイグレーション

## 本番用データベースの選択肢

| サービス | 特徴 | 無料枠 |
|---------|------|--------|
| **Vercel Postgres** | Vercelと統合、設定簡単 | あり |
| **Supabase** | 機能豊富、管理画面が使いやすい | あり |
| **Neon** | サーバーレス、コスパ良い | あり |
| **PlanetScale** | MySQLベース、ブランチ機能 | あり |

この本では **Vercel Postgres** を使います（Vercelとの相性が良いため）。

## Vercel Postgres のセットアップ

1. Vercelダッシュボードで「Storage」タブを選択
2. 「Create Database」→「Postgres」
3. データベース名を入力して作成
4. 接続情報が自動で環境変数に設定される

## マイグレーションの実行

本番データベースにテーブルを作成します。

```bash
npx prisma migrate deploy
```

## この章のまとめ

- 本番用データベースには複数の選択肢がある
- Vercel Postgres は設定が簡単
- マイグレーションで本番DBにテーブルを作成
