## データベースとは

**データベース**は、データを整理して保存する仕組みです。

localStorageとの違い：

| 項目 | localStorage | データベース |
|-----|-------------|-------------|
| 保存場所 | ブラウザ | サーバー |
| 容量 | 5MB程度 | 無制限（実質） |
| 共有 | できない | 複数ユーザーで共有 |
| 検索 | 遅い | 高速 |
| セキュリティ | 低い | 高い |

## リレーショナルデータベース（RDB）

データを**テーブル**（表）で管理するデータベースです。

Excel のような表形式でデータを保存します。

```
competitions テーブル
┌────┬──────────────┬──────────┐
│ id │ name         │ status   │
├────┼──────────────┼──────────┤
│ 1  │ 春季大会      │ ongoing  │
│ 2  │ 秋季大会      │ upcoming │
│ 3  │ 冬季大会      │ completed│
└────┴──────────────┴──────────┘
```

### 用語

| 用語 | 説明 | Excelで言うと |
|-----|------|--------------|
| テーブル | データの種類ごとの表 | シート |
| 行（レコード） | 1件のデータ | 行 |
| 列（カラム） | データの項目 | 列 |
| 主キー（Primary Key） | 各行を一意に識別する列 | - |

## テーブルの関連（リレーション）

複数のテーブルを関連付けて使います。

```
competitions（大会）
┌────┬──────────────┐
│ id │ name         │
├────┼──────────────┤
│ 1  │ 春季大会      │
└────┴──────────────┘
         │
         │ competition_id で関連付け
         ↓
teams（チーム）
┌────┬──────────────┬────────────────┐
│ id │ name         │ competition_id │
├────┼──────────────┼────────────────┤
│ 1  │ FCレッドスター │ 1              │
│ 2  │ ブルーウィングス│ 1              │
└────┴──────────────┴────────────────┘
```

`competition_id` で、どの大会に所属するチームかを表しています。

## 主なリレーションの種類

### 1対多（One to Many）

1つの大会に、複数のチームが所属する。

```
Competition (1) ──────── (多) Team
```

### 多対多（Many to Many）

1つの試合に、2つのチームが参加する。
1つのチームが、複数の試合に参加する。

```
Team (多) ──────── (多) Match
```

多対多の場合、中間テーブルを使います。

## SQL の基本概念

SQLは、データベースを操作するための言語です。

### 主な操作

| 操作 | SQL | 説明 |
|-----|-----|------|
| 取得 | SELECT | データを読み取る |
| 作成 | INSERT | データを追加する |
| 更新 | UPDATE | データを変更する |
| 削除 | DELETE | データを消す |

### 例

```sql
-- 大会一覧を取得
SELECT * FROM competitions;

-- 新しい大会を作成
INSERT INTO competitions (name, status) VALUES ('春季大会', 'upcoming');

-- 大会を更新
UPDATE competitions SET status = 'ongoing' WHERE id = 1;

-- 大会を削除
DELETE FROM competitions WHERE id = 1;
```

:::message
「Next.jsでバックエンド連携＆サービス公開」のPart 2では **Prisma** という ORM を使うので、SQL を直接書くことは少ないです。
ただし、概念を知っておくとデバッグに役立ちます。
:::

## ORM（Object-Relational Mapping）

SQLを直接書かずに、プログラミング言語でデータベースを操作する仕組みです。

SQLで書く場合：
```sql
SELECT * FROM competitions WHERE status = 'ongoing';
```

Prisma（ORM）で書く場合：
```typescript
const competitions = await prisma.competition.findMany({
  where: { status: 'ongoing' }
});
```

JavaScriptのコードでデータベースを操作できるので、開発しやすくなります。

## この章のまとめ

- データベースはデータを整理して保存する仕組み
- リレーショナルデータベースはテーブル（表）でデータを管理
- テーブル同士を関連付けて複雑なデータを表現
- ORM を使うと SQL を書かずにデータベースを操作できる
