## この本について

この本は、HTML/CSSは触ったことがあるけど、**Webサービスを作ったことがない方**向けの入門書です。

「Reactを学びたいけど、何から始めればいいかわからない」
「プログラミングスクールに通うほどではないけど、独学だと挫折しそう」

そんな方に向けて、**最小限の知識で実際に手を動かせる**ところまでガイドします。

## 3部構成の教材について

React/Next.js の学習は、**3部構成**の教材で段階的に進めます。

| 編 | 教材名 | 対象 | 内容 |
|---|--------|------|------|
| **第1編** | **HTML/CSSの次はこれ！React & Next.js 超入門**<br>（この本） | 超初心者 | Reactの基本概念、useState、Next.jsの基本操作 |
| **第2編** | **React Hooksと実践パターン** | 中級者 | useEffect、条件付きレンダリング、フォーム処理、データフェッチング |
| **第3編** | **Reactで作る！スポーツ大会管理サービス** | 実践者 | 実際のアプリを作りながら、CRUD操作、認証、デプロイを学ぶ |

### この本（第1編）で扱うこと

**Reactの基礎**
- コンポーネント、props、state の概念
- JSX の書き方
- `useState` フック（状態管理の基本）

**Next.jsの基礎**
- プロジェクトの作成と起動
- ページの追加方法
- `Link` コンポーネント

**開発環境**
- Node.js、VSCode のセットアップ
- Git の基本操作
- TypeScript の基礎

### この本で扱わないこと（第2編・第3編で学びます）

**第2編で学ぶ内容**
- `useEffect` フック（副作用処理、データ取得）
- 条件付きレンダリング（if文、三項演算子）
- フォーム処理とバリデーション
- データフェッチング（API呼び出し）
- エラーハンドリングとローディング状態

**第3編で学ぶ内容**
- 実際のアプリ開発（CRUD操作）

## この本のゴール

この本を読み終えると、以下ができるようになります。

- 開発環境（Node.js, VSCode）が整っている
- Gitでコードを管理できる
- Reactの基本概念（コンポーネント、props、state）を理解している

## 対象読者

- HTML/CSSは触ったことがある
- JavaScriptは少しだけ知っている（または知らなくてもOK）
- Webサービスを作ったことはない
- Gitを使ったことがない（または使い方がよくわからない）

## この本で扱わないこと

以下は次のステップとして別の本で扱います。

- GitHub へのコードのアップロード（push/pull）
- データベース連携
- 認証・認可（ログイン機能など）
- サーバーへの公開（デプロイ）

## 学習の進め方

この本を効果的に学ぶためのアドバイスです。

### 手を動かしながら読む

コードを読むだけでなく、**実際に書いて動かす**ことが大切です。

- 各章のコードを写経（書き写す）してみる
- 少し変えてみて、どう変わるか確認する
- エラーが出たら、エラーメッセージを読んで解決してみる

### わからないことがあったら

1. **エラーメッセージを読む**
   - エラーメッセージには、問題のヒントが書かれています
   - 最初は難しく感じますが、慣れると読めるようになります

2. **公式ドキュメントを確認する**
   - [React 公式ドキュメント](https://react.dev/)
   - [Next.js 公式ドキュメント](https://nextjs.org/docs)
   - [TypeScript 公式ドキュメント](https://www.typescriptlang.org/docs/)

3. **エラーメッセージで検索する**
   - エラーメッセージをコピーして Google で検索
   - 同じ問題に遭遇した人の解決策が見つかることが多い

4. **少し休む**
   - 詰まったら一度休憩する
   - 頭をリフレッシュすると、解決策が見つかることがあります

### 完璧を目指さない

最初から完璧に理解しようとしなくて大丈夫です。

- わからない部分は「後で調べる」とメモしておく
- 全体を一度通して読んでから、わからない部分を深掘りする
- 使っていくうちに理解が深まります

### コードを写経するときの注意点

- **タイポ（スペルミス）に注意**
  - 大文字・小文字を間違えない
  - 括弧やセミコロンを忘れない

- **インデント（字下げ）を正しく**
  - VSCode が自動で整えてくれますが、手動で書くときは注意

- **エラーが出たら慌てない**
  - エラーは学習のチャンスです
  - エラーメッセージを読んで、何が問題か考えてみましょう
