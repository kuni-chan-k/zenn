## この章でやること

- VSCode の便利な機能を知る
- 開発効率を上げる設定を覚える
- エラーの確認方法を学ぶ

## 自動保存の設定

ファイルを編集するたびに手動で保存するのは大変です。自動保存を有効にしましょう。

### 設定方法

1. VSCode の左下の歯車アイコン（⚙️）をクリック
2. 「設定」を選択
3. 検索ボックスに「自動保存」と入力
4. 「Files: Auto Save」を「afterDelay」に設定

これで、ファイルを編集して数秒経つと自動で保存されます。

:::message
設定画面は `Cmd + ,`（Mac）または `Ctrl + ,`（Windows）でも開けます。
:::

## エラーの確認方法

TypeScript や ESLint のエラーは、VSCode が自動で見つけてくれます。

### 問題パネルの開き方

1. 画面下部の「問題」タブをクリック
2. または `Cmd + Shift + M`（Mac）/ `Ctrl + Shift + M`（Windows）

エラーがあると、赤い波線が表示され、問題パネルに一覧が表示されます。

### エラーの見方

```
✗ 'count' is assigned a value but never used.
  → count という変数が使われていない
```

エラーメッセージをクリックすると、該当箇所に移動できます。

## コードフォーマット（Prettier）

コードの見た目を自動で整える機能です。Prettier 拡張機能をインストールしていれば使えます。

### 使い方

1. ファイルを開く
2. `Shift + Option + F`（Mac）/ `Shift + Alt + F`（Windows）
3. または右クリック → 「ドキュメントのフォーマット」

コードが自動で整形されます。

### 保存時に自動フォーマット

保存するたびに自動でフォーマットする設定：

1. 設定を開く（`Cmd + ,` / `Ctrl + ,`）
2. 「format on save」で検索
3. 「Editor: Format On Save」にチェック

## コード補完

VSCode は、コードを入力しているときに候補を表示してくれます。

### 使い方

1. コードを入力すると、候補が自動で表示される
2. `↑` `↓` キーで選択
3. `Enter` または `Tab` で確定

例：`use` と入力すると、`useState` などの候補が表示されます。

### インポートの自動補完

ファイル名を入力すると、自動でインポート文を生成してくれます。

```tsx
// Header と入力すると...
import Header from './components/Header';
// が自動で追加される
```

## ファイル検索

プロジェクト内のファイルを素早く開けます。

### 使い方

- `Cmd + P`（Mac）/ `Ctrl + P`（Windows）
- ファイル名を入力して検索
- `Enter` で開く

例：`page.tsx` と入力すると、該当ファイルが表示されます。

## 統合ターミナル

VSCode の中にターミナルを表示できます。

### 開き方

- `Ctrl + @`（Mac）/ `Ctrl + @`（Windows）
- または「表示」→「ターミナル」

ターミナルが画面下部に表示されます。`npm run dev` などのコマンドを実行できます。

### 複数のターミナル

「+」ボタンをクリックすると、複数のターミナルを開けます。開発サーバーと Git コマンドを別々に実行するときに便利です。

## コードの検索・置換

ファイル内やプロジェクト全体で文字列を検索・置換できます。

### ファイル内の検索

- `Cmd + F`（Mac）/ `Ctrl + F`（Windows）
- 検索したい文字列を入力

### プロジェクト全体の検索

- `Cmd + Shift + F`（Mac）/ `Ctrl + Shift + F`（Windows）
- 検索したい文字列を入力
- 検索結果が一覧表示される

### 置換

検索画面で「置換」アイコンをクリックすると、文字列を一括で置き換えられます。

:::message alert
プロジェクト全体の置換は、間違えると大変なことになるので、慎重に行いましょう。まずは1つのファイルで試すことをおすすめします。
:::

## コードの折りたたみ

長いコードを見やすくする機能です。

### 使い方

1. 行番号の左側に `▼` マークが表示される
2. クリックすると、そのブロックが折りたたまれる
3. もう一度クリックすると展開される

関数やコンポーネントを折りたたんで、全体を見やすくできます。

## マルチカーソル

複数の場所を同時に編集できます。

### 使い方

1. `Cmd + Option + ↑`（Mac）/ `Ctrl + Alt + ↑`（Windows）でカーソルを上に追加
2. `Cmd + Option + ↓`（Mac）/ `Ctrl + Alt + ↓`（Windows）でカーソルを下に追加
3. 同じ文字列を複数選択する場合は、`Cmd + D`（Mac）/ `Ctrl + D`（Windows）

例：複数の `div` を同時に `section` に変更するときなどに便利です。

## スニペット（コードのテンプレート）

よく使うコードをテンプレートとして登録できます。

### React コンポーネントのスニペット

`ES7+ React/Redux/React-Native snippets` 拡張機能をインストールしている場合：

1. `.tsx` ファイルで `rafce` と入力
2. `Tab` を押す
3. 関数コンポーネントのテンプレートが挿入される

```tsx
// rafce と入力して Tab
export const ComponentName = () => {
  return <div>ComponentName</div>;
};
```

### よく使うスニペット

| 入力 | 結果 |
|------|------|
| `rafce` | 関数コンポーネント（export） |
| `rafc` | 関数コンポーネント（exportなし） |
| `useState` | useState フック |
| `useEffect` | useEffect フック |

よく使うショートカットをまとめました。

|------|-----|---------|
| 設定を開く | `Cmd + ,` | `Ctrl + ,` |
| ファイル検索 | `Cmd + P` | `Ctrl + P` |
| プロジェクト内検索 | `Cmd + Shift + F` | `Ctrl + Shift + F` |
| ターミナルを開く | `Ctrl + @` | `Ctrl + @` |
| コードフォーマット | `Shift + Option + F` | `Shift + Alt + F` |
| 問題パネル | `Cmd + Shift + M` | `Ctrl + Shift + M` |
| ファイルを閉じる | `Cmd + W` | `Ctrl + W` |
| ファイルを再開 | `Cmd + Shift + T` | `Ctrl + Shift + T` |

- 問題パネルでエラーを確認できる

- 統合ターミナルでコマンドを実行できる
- ショートカットを覚えると作業が速くなる
