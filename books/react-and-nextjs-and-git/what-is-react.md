## React とは

React は、**Webサイトの見た目（UI）を作るためのライブラリ**です。Facebook（現Meta）が開発し、世界中で広く使われています。

「ライブラリ」とは、便利な機能をまとめたプログラムの集まりのことです。React を使うと、複雑なWebサイトを効率的に作れます。

## なぜ React を使うのか

### 従来の方法の問題点

HTML と JavaScript で Web サイトを作る場合、こんなコードになります。

```html
<button id="counter-btn">クリック: 0</button>

<script>
  let count = 0;
  const button = document.getElementById('counter-btn');
  button.addEventListener('click', () => {
    count++;
    button.textContent = `クリック: ${count}`;
  });
</script>
```

小さなアプリならこれでも大丈夫ですが、規模が大きくなると問題が出てきます。

- どの要素を、いつ、どう更新するか管理が大変
- 同じような部品を何度も作り直す
- コードが複雑になり、バグが増える

### React が解決すること

React を使うと、同じ機能がこう書けます。

```jsx
function Counter() {
  const [count, setCount] = useState(0);

  return (
    <button onClick={() => setCount(count + 1)}>
      クリック: {count}
    </button>
  );
}
```

React の特徴は以下の通りです。

1. **コンポーネント** - 部品として再利用できる
2. **宣言的** - 「今どうあるべきか」を書くだけで、React が更新を処理
3. **効率的** - 必要な部分だけを更新する

## コンポーネント思考

React の最も重要な概念が「**コンポーネント**」です。

### コンポーネントとは

コンポーネントは、**画面の一部分を担当する部品**です。

たとえば、このようなWebページがあったとします。

```
┌─────────────────────────────┐
│        ヘッダー              │
├─────────────────────────────┤
│ サイドバー │    メイン       │
│            │                │
│  ・リンク1  │   記事一覧     │
│  ・リンク2  │   ┌───┐ ┌───┐ │
│  ・リンク3  │   │カード│ │カード│ │
│            │   └───┘ └───┘ │
├─────────────────────────────┤
│        フッター              │
└─────────────────────────────┘
```

これを React のコンポーネントで表すと、こうなります。

```jsx
function App() {
  return (
    <div>
      <Header />
      <div>
        <Sidebar />
        <Main>
          <ArticleCard />
          <ArticleCard />
        </Main>
      </div>
      <Footer />
    </div>
  );
}
```

### コンポーネントのメリット

1. **再利用できる** - 同じ部品を何度も使える（例: ArticleCard）
2. **分担できる** - 部品ごとに開発できる
3. **理解しやすい** - 構造が見てわかる
4. **修正しやすい** - 1箇所直せば全部に反映

## 仮想 DOM

React のもう一つの特徴が「**仮想 DOM**」です。

### DOM とは

DOM（Document Object Model）は、HTML をプログラムから操作するための仕組みです。

```javascript
// DOM を直接操作する例
document.getElementById('title').textContent = '新しいタイトル';
```

DOM の操作は「重い」処理で、頻繁に行うとページが遅くなります。

### 仮想 DOM の仕組み

React は「仮想 DOM」という仕組みを使います。

1. React はメモリ上に「仮想的な DOM」を持っている
2. 状態が変わると、まず仮想 DOM を更新
3. 前の仮想 DOM と比較して、変更点を特定
4. 実際の DOM は、変更があった部分だけ更新

これにより、無駄な更新を減らし、高速に動作します。

:::message
仮想 DOM の仕組みを完全に理解する必要はありません。「React が賢く画面を更新してくれる」と覚えておけばOKです。
:::
