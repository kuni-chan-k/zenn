## React とは

React は、**Webサイトの見た目（UI）を作るためのライブラリ**です。2013年に Facebook（現Meta）が公開し、現在では世界で最も人気のある UI ライブラリになりました。

「ライブラリ」とは、便利な機能をまとめたプログラムの集まりのことです。React を使うと、複雑なWebサイトを効率的に作れます。

## 世界中で使われている React

React は、あなたが普段使っているサービスの多くで採用されています。

### 有名サービスの採用例

| サービス | 説明 |
|---------|------|
| **Facebook / Instagram** | React を開発した Meta 社のサービス |
| **Netflix** | 動画ストリーミングサービス |
| **Airbnb** | 宿泊予約サービス |
| **Uber** | 配車サービス |
| **Discord** | コミュニケーションツール |
| **Notion** | ドキュメント・タスク管理 |
| **Shopify** | ECプラットフォーム |

これらのサービスに共通しているのは、**複雑な UI を持つ大規模なアプリケーション**であることです。React は、こうした複雑なアプリを効率よく開発・運用するために選ばれています。

### なぜこれほど人気なのか

React が世界中で選ばれる理由は以下の通りです。

1. **学習コストと生産性のバランス**
   - シンプルな概念（コンポーネント、props、state）を覚えれば始められる
   - 複雑なアプリでも同じ考え方で構築できる

2. **巨大なエコシステム**
   - 数十万のライブラリやツールが利用可能
   - 困ったときに解決策が見つかりやすい

3. **活発なコミュニティ**
   - 世界中に開発者がいて、情報が豊富
   - 日本語の学習リソースも充実

4. **求人市場での需要**
   - フロントエンド開発の求人で最も需要が高い
   - スタートアップから大企業まで幅広く採用

:::message
React を学ぶことは、単に一つのライブラリを覚えるだけでなく、**現代のフロントエンド開発のスタンダード**を身につけることを意味します。
:::

## なぜ React を使うのか

### 従来の方法の問題点

HTML と JavaScript で Web サイトを作る場合、こんなコードになります。

```html
<button id="counter-btn">クリック: 0</button>

<script>
  let count = 0;
  const button = document.getElementById('counter-btn');
  button.addEventListener('click', () => {
    count++;
    button.textContent = `クリック: ${count}`;
  });
</script>
```

小さなアプリならこれでも大丈夫ですが、規模が大きくなると問題が出てきます。

- どの要素を、いつ、どう更新するか管理が大変
- 同じような部品を何度も作り直す
- コードが複雑になり、バグが増える

### React が解決すること

React を使うと、同じ機能がこう書けます。

```jsx
function Counter() {
  const [count, setCount] = useState(0);

  return (
    <button onClick={() => setCount(count + 1)}>
      クリック: {count}
    </button>
  );
}
```

React の特徴は以下の通りです。

1. **コンポーネント** - 部品として再利用できる
2. **宣言的** - 「今どうあるべきか」を書くだけで、React が更新を処理
3. **効率的** - 必要な部分だけを更新する

## コンポーネント思考

React の最も重要な概念が「**コンポーネント**」です。

### コンポーネントとは

コンポーネントは、**画面の一部分を担当する部品**です。

たとえば、このようなWebページがあったとします。

```
┌─────────────────────────────┐
│        ヘッダー              │
├─────────────────────────────┤
│ サイドバー │    メイン       │
│            │                │
│  ・リンク1  │   記事一覧     │
│  ・リンク2  │   ┌───┐ ┌───┐ │
│  ・リンク3  │   │カード│ │カード│ │
│            │   └───┘ └───┘ │
├─────────────────────────────┤
│        フッター              │
└─────────────────────────────┘
```

これを React のコンポーネントで表すと、こうなります。

```jsx
function App() {
  return (
    <div>
      <Header />
      <div>
        <Sidebar />
        <Main>
          <ArticleCard />
          <ArticleCard />
        </Main>
      </div>
      <Footer />
    </div>
  );
}
```

### コンポーネントのメリット

1. **再利用できる** - 同じ部品を何度も使える（例: ArticleCard）
2. **分担できる** - 部品ごとに開発できる
3. **理解しやすい** - 構造が見てわかる
4. **修正しやすい** - 1箇所直せば全部に反映

## React を使う5つのメリット

ここまでの内容をまとめつつ、React を使うメリットを整理します。

### 1. 再利用可能なコンポーネント

一度作った部品は、何度でも使い回せます。

```tsx
// ボタンを一度作れば...
function Button({ text }: { text: string }) {
  return <button className="btn">{text}</button>;
}

// どこでも使える
<Button text="送信" />
<Button text="キャンセル" />
<Button text="削除" />
```

チーム開発では「UIコンポーネントライブラリ」として共有し、デザインの統一と開発効率の向上を実現できます。

### 2. 予測しやすいデータフロー

React では、データは**親から子へ一方向**に流れます。

```
親コンポーネント
    ↓ props
子コンポーネント
    ↓ props
孫コンポーネント
```

この仕組みにより、「どこでデータが変わったか」が追跡しやすく、バグの原因を特定しやすくなります。

### 3. 豊富なツールとライブラリ

React のエコシステムには、あらゆる用途のツールが揃っています。

| 用途 | ライブラリ例 |
|------|-------------|
| ルーティング | React Router, Next.js |
| 状態管理 | Redux, Zustand, Jotai |
| フォーム | React Hook Form, Formik |
| UI コンポーネント | shadcn/ui, MUI, Chakra UI |
| アニメーション | Framer Motion, React Spring |
| データ取得 | TanStack Query, SWR |

必要な機能を組み合わせて、自分のプロジェクトに最適な構成を作れます。

### 4. 学んだスキルが活かせる

React の考え方は、他の技術にも応用できます。

- **React Native** → スマホアプリ開発（iOS / Android）
- **Electron** → デスクトップアプリ開発
- **Next.js** → フルスタック Web 開発

React を学べば、Web だけでなく、さまざまなプラットフォームでアプリを作れるようになります。

### 5. 長期的なサポート

React は Meta 社が開発・維持しており、今後も長期的にサポートされます。

- 2013年のリリースから10年以上の実績
- 後方互換性を重視したアップデート
- 大企業での採用実績による信頼性

「学んだ技術がすぐに使えなくなる」という心配が少ないのも、React を選ぶ大きな理由です。

## 仮想 DOM

React のもう一つの特徴が「**仮想 DOM**」です。

### DOM とは

DOM（Document Object Model）は、HTML をプログラムから操作するための仕組みです。

```javascript
// DOM を直接操作する例
document.getElementById('title').textContent = '新しいタイトル';
```

DOM の操作は「重い」処理で、頻繁に行うとページが遅くなります。

### 仮想 DOM の仕組み

React は「仮想 DOM」という仕組みを使います。

1. React はメモリ上に「仮想的な DOM」を持っている
2. 状態が変わると、まず仮想 DOM を更新
3. 前の仮想 DOM と比較して、変更点を特定
4. 実際の DOM は、変更があった部分だけ更新

これにより、無駄な更新を減らし、高速に動作します。

:::message
仮想 DOM の仕組みを完全に理解する必要はありません。「React が賢く画面を更新してくれる」と覚えておけばOKです。
:::
